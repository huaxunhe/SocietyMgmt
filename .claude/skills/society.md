## App Context

**Purpose**: Society & Apartment Management System (India-Focused). Replaces WhatsApp groups and Excel sheets with a proper ServiceNow app.

### Core Tables

| Table | Description |
|-------|-------------|
| `x_664892_society_0_flat` | Master flat/unit registry — all other tables reference this |
| `x_664892_society_0_maintenance_bill` | One record per flat per month, auto-generated by scheduled job |
| `x_664892_society_0_service_request` | Unified table for complaints, facility bookings, and notices. Distinguished by `request_type` choice field |
| `x_664892_society_0_visitor_log` | Gate entry log maintained by security staff |

### Server-Side Automation

**Business Rules**:
- `set-dates`: Auto-fills `opened_on` on insert, `resolved_on` on resolve
- `prevent-double-booking`: Aborts facility booking if slot already taken

**Script Includes**:
- `SocietyUtils`: `isSlotAvailable()`, `getUnpaidBillsByFlat()`, `getOpenRequestsByType()`
- `MaintenanceBillingEngine`: `generateMonthlyBills()`, `markOverdueBills()`

**Scheduled Jobs**:
- Daily @ 1AM: Marks unpaid past-due bills as overdue
- Monthly @ 1st: Auto-creates bill records for all active flats
